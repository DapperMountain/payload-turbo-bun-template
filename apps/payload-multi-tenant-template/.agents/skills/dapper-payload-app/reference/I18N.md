# App translations and localization

## Not the Payload monorepo workflow

This app does **not** use Payload core’s `generate-translations` tooling:

- No `packages/translations` or `tools/scripts` in this repo
- No `pnpm generateTranslations:core` / `generateTranslations:plugin-*`

That workflow is for contributing to [payloadcms/payload](https://github.com/payloadcms/payload), not for this template.

## Two layers in this app

| Layer | Location | Purpose |
|-------|----------|---------|
| **Payload admin UI** | `@payloadcms/translations` (en, es) | Built-in admin strings |
| **App copy** | `src/lang/en.ts`, `src/lang/es.ts` | Roles labels, frontend, meta |

Wiring: `src/lang/index.ts` exports `i18n` and `localization` consumed by `config/payload.ts`.

## Folder layout

| Path | Contents |
|------|----------|
| `src/lang/en.ts`, `es.ts` | Locale bundle (`export default { custom: { … } }`) — shape matches Payload merge |
| `src/lang/types.ts` | `CustomTranslationsObject`, `CustomTranslationKeys` (`NestedKeysStripped`) |
| `src/lang/index.ts` | `locales` array → `supportedLanguages`, `i18n.translations`, `localization` |
| `src/utils/i18n.client.ts` | `useAppTranslation()` — typed client hook |
| `src/utils/i18n.server.ts` | `getRequestI18n()` — Server Components only (`server-only`) |

Each locale file exports the **exact object** merged into `i18n.translations[code]` — including the `custom` key ([Payload custom translations](https://payloadcms.com/docs/configuration/i18n#custom-translations)):

```typescript
// src/lang/en.ts
export default {
  custom: {
    frontend: { welcome: 'Welcome' },
    meta: { title: 'Payload' },
  },
} as const
```

Runtime keys mirror that shape: `t('custom:frontend:welcome')`. Payload admin strings use their own namespaces in the same `t()` (e.g. `t('general:save')`).

## Three problems (do not mix them)

| Problem | Mechanism | Example in this app |
|---------|-----------|---------------------|
| Admin UI + runtime `t()` | `config.i18n` + `custom:*` or Payload namespaces | Frontend copy (`custom:frontend:*`) |
| Localized **document** fields | `localization` + `localized: true` | Multilingual entry content in the DB |
| **Config-time** labels / options | Plain strings or `custom` from `@/lang` | `Users` role option labels in collection config |

`t('custom:…')` and **`import { custom } from '@/lang'`** use the same key tree — config-time reads English only, not the user’s cookie-selected language.

## Server vs client (Next App Router)

Reference implementation: `src/app/(frontend)/`.

```text
layout.tsx (Server)
  getRequestI18n()
  <html lang={language}>
  FrontendProviders (Client) — TranslationProvider + DesignSystemProvider (next)
    page.tsx (Server) — getPayload, auth (data only)
      HomePage (Client) — useAppTranslation() → t('custom:frontend:*')
actions/switch-language.ts (Server Action) — sets payload-lng cookie
```

| File | Layer | Responsibility |
|------|-------|----------------|
| `layout.tsx` | Server | `getRequestI18n()`; set `<html lang>`; pass i18n into `TranslationProvider` |
| `_components/providers.tsx` | Client | `TranslationProvider` + `DesignSystemProvider` |
| `page.tsx` | Server | `getPayload`, auth — **no copy** |
| `_components/home-page.tsx` | Client | `useAppTranslation()` + `t('custom:…')` |
| `actions/switch-language.ts` | Server Action | Set `${cookiePrefix}-lng` cookie |
| `_components/language-switcher.tsx` | Client | `<select>` + `router.refresh()` |

Payload docs: [Custom components — getting the current language](https://payloadcms.com/docs/custom-components/overview#getting-the-current-language).

### One translation pipeline

| Layer | API | Example |
|-------|-----|---------|
| **Server** (RSC, metadata) | `getRequestI18n()` from `@/utils/i18n.server` | `t('custom:meta:title')` |
| **Client** | `useAppTranslation()` from `@/utils/i18n.client` | `t('custom:frontend:welcome')` |
| **Types** | `CustomTranslationKeys` from `@/lang/types` | Autocomplete + no `as never` |

```typescript
// Server Component
import { getRequestI18n } from '@/utils/i18n.server'

const { t } = await getRequestI18n()
t('custom:meta:title')

// Client Component
import { useAppTranslation } from '@/utils/i18n.client'

const { t } = useAppTranslation()
t('custom:frontend:welcome')
```

### Config-time collection labels

- Use **`import { custom } from '@/lang'`** for shared enums (e.g. `custom.roles`) — see `src/collections/Users/index.ts`.
- For labels that must follow the **admin language switch**, use Payload’s localized `label` objects or `t()` in admin components.

## Adding or changing strings

1. Edit **`src/lang/en.ts`** under `custom` (nested groups like `frontend`, `meta`, `roles`).
2. Mirror keys in **`src/lang/es.ts`**.
3. To add a **new locale**: add `src/lang/<code>.ts` (same `{ custom: { … } }` shape) and one row in the `locales` array in `src/lang/index.ts`.
4. Use runtime keys as **`custom:<group>:<key>`** matching the object shape (e.g. `custom:frontend:welcome`).

## Frontend language switch

`src/app/(frontend)/actions/switch-language.ts` — server action sets Payload language cookie (`${cookiePrefix}-lng`).
